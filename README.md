# skrra.dev

Personal developer portfolio for **Christoffer Lööf** — system developer student focused on testing, QA, automation, Java and Minecraft/Fabric modding. Live at [skrra.dev](https://skrra.dev).

The site is designed as a dark "developer lab / GitHub dashboard" rather than a traditional portfolio: a terminal-style hero, live GitHub statistics, case-study project cards and an activity feed.

## Tech stack

- [Next.js 16](https://nextjs.org) (App Router, TypeScript)
- [Tailwind CSS v4](https://tailwindcss.com) (CSS-first config via `@theme`)
- [lucide-react](https://lucide.dev) for icons
- Custom SVG donut chart (no chart library)
- Deployed on Vercel

## Features

- **Terminal hero** — syntax-highlighted `whoami` card with line numbers
- **Featured projects** — curated project data merged with live GitHub metadata (stars, forks, last push), with expandable full-width case-study panels
- **GitHub dashboard** — profile stats, most-used languages chart and a recent activity feed built from the GitHub REST API
- **Timeline** — education & experience with status badges
- **Fully static** — the page is prerendered and revalidated hourly (ISR)

## GitHub integration

All GitHub data is fetched **server-side** in [`lib/github.ts`](lib/github.ts):

- Uses the public REST API (`/users`, `/repos`, `/events/public`)
- Responses are cached for one hour (`next: { revalidate: 3600 }`) to stay fast and avoid rate limits
- An optional `GITHUB_TOKEN` raises the rate limit; it is only read on the server and never shipped to the client
- If the API is unreachable, the site falls back to neutral placeholder data instead of breaking — no invented numbers, and the dashboard notes that live data could not be refreshed

## Getting started

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### Environment variables

Copy `.env.example` to `.env.local`:

| Variable                   | Required | Description                                                        |
| -------------------------- | -------- | ------------------------------------------------------------------ |
| `GITHUB_USERNAME`          | No       | GitHub account to display (defaults to Stoffe101)                   |
| `GITHUB_TOKEN`             | No       | Personal access token for a higher API rate limit                   |
| `RESEND_API_KEY`           | No\*     | Resend API key for the contact form (server-side only)              |
| `RESEND_FROM`              | No       | Verified sender, defaults to `skrra.dev Kontakt <noreply@skrra.dev>` |
| `GOOGLE_SITE_VERIFICATION` | No       | Google Search Console verification token (tag omitted if unset)     |

\* Without `RESEND_API_KEY` the contact form shows a controlled error with a mailto fallback — the page still builds and works.

## Contact form (Resend)

The contact form posts through a Next.js server action ([`app/actions/send-contact.ts`](app/actions/send-contact.ts)) that sends mail via [Resend](https://resend.com) — the API key never reaches the client. Spam protection: honeypot field, server-side validation with length limits, HTML escaping and a best-effort per-IP rate limit.

Setup:

1. Create a Resend account and verify the sending domain (e.g. `skrra.dev`)
2. Set `RESEND_API_KEY` (and optionally `RESEND_FROM`) in `.env.local` / Vercel project settings

## Google Search Console

1. Add `skrra.dev` as a property in [Google Search Console](https://search.google.com/search-console) — preferably verified via a DNS TXT record for the domain (alternatively set `GOOGLE_SITE_VERIFICATION` for meta-tag verification)
2. Submit the sitemap: `https://skrra.dev/sitemap.xml`
3. Use URL Inspection on `https://skrra.dev` to request indexing

`robots.txt` and `sitemap.xml` are generated by [`app/robots.ts`](app/robots.ts) and [`app/sitemap.ts`](app/sitemap.ts).

## Analytics

Vercel Web Analytics and Speed Insights are enabled via `@vercel/analytics` and `@vercel/speed-insights` in the root layout — anonymous, cookie-free metrics only.

## CV generation

`public/cv.pdf` is generated from an HTML source — **do not edit the PDF directly**. To update the CV:

1. Edit [`scripts/cv/cv.html`](scripts/cv/cv.html)
2. Install Playwright locally if needed: `npm install -D playwright --no-save` (then `npx playwright install chromium`)
3. Regenerate: `node scripts/cv/generate-pdf.js`

The generator renders the HTML headlessly and exports a single full-bleed A4 page with clickable links.
